null vs undefined

In JavaScript there are only six falsy values. Both null and undefined are two of the six falsy values. Here’s a full list:
false
0 (zero)
“” (empty string)
null
undefined
NaN (Not A Number)

Also in JavaScript, there are six primitive values. Both null and undefined are primitive values. Here is a full list:
Boolean
Null
Undefined
Number
String
Symbol

null !== undefined (no strict equivalence)
null == undefined (loose equivalence)

Null has one value: null. It is explicitly nothing.
Undefined - is a variable that has no value.

setting a value to Null is on purpose, while undefined may occur unintentionally

Summary
null is an assigned value. It means nothing.
undefined typically means a variable has been declared but not defined yet.
null and undefined are falsy values.
null and undefined are both primitives. However an error shows that typeof null = object.
null !== undefined but null == undefined.
